## 涉及的高風險改動

- **`polaroid-carousel.tsx`**：每張 card DOM 節點均設 `willChange: 'transform'`，最多 5 層疊加，GSAP 同時驅動 rotation / scale / y / zIndex — 在低階行動裝置上可能升高 compositor 記憶體
- **`cinema-gallery.tsx`**：ghost image 使用 `filter: blur(4px) brightness(0.2)` 並於 GSAP timeline 中更換 src，blur filter 在行動裝置 GPU 上消耗顯著；主圖 `willChange: 'transform, opacity'` 搭配三張圖同時保留在 DOM
- **`read-page-client.tsx`**：在 read page 核心渲染路徑中引入 `PolaroidCarousel`，影響所有 `type === 'carousel'` 頁面的 FPS 表現

## 建議

建議在提交前於真實裝置（尤其 mid-range Android）執行以下驗證，並將結果存入 `.perf-log/`：

1. **Chrome DevTools → Performance**：錄製一次 PolaroidCarousel shuffle 動畫，確認無 long frame（> 16ms）
2. **Chrome DevTools → Layers**：檢查 `willChange` 層數是否合理，避免 layer explosion
3. **CinemaGallery**：在行動裝置上左右滑動切換圖片，觀察是否有卡頓或 jank

建議的 commit message：

```
feat(read): 新增 PolaroidCarousel 與 CinemaGallery 元件

[perf-needed: /read (carousel type pages)]
```
