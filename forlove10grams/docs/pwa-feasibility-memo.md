# PWA 可行性備忘 — 疑慮與分層評估

> 狀態：探索記錄，未決定實作
> 日期：2026-07-07

## 背景

在探索垂直客群（公益團隊、鋼琴家教、潛水教練）時，浮現「離線環境記錄」的需求——典型場景是潛水教練在船上無網路，想趁記憶新鮮記下每個學生的狀況。本備忘記錄「轉為 PWA」的技術疑慮，供日後決策參考。

核心結論先講：**「轉 PWA」不是一個決定，是四層深度不同的工程**，難度從一天到「PWA 根本做不到」都有。真正的需求是「離線寫入」（第三層），而最直覺想做的「離線閱讀」（第二層）反而價值最低、且與本專案的媒體保護機制直接相剋。

---

## 四層難度

### 第一層：可安裝（1–2 天）

Manifest、icon、service worker 註冊（Next.js 生態用 Serwist 等套件）。完成後可「加到主畫面」、全螢幕開啟。

- 本質是化妝，離線能力趨近於零
- 但它是門票：**iOS 上「已安裝」的 PWA 才豁免 Safari 的 7 天閒置清除儲存規則**，後面任何一層都以此為前提
- 隨時可做，無依賴

### 第二層：離線快取閱讀（數週，且與現有架構相剋）

一般網站套公式即可，本專案有三個特有地雷：

1. **簽章媒體與快取相剋**。圖片是每天 UTC 午夜到期的 Signed URL——快取舊 URL 隔天變一批 403 破圖；URL 天天換又讓快取天天 miss。影片更難：HLS 靠 Signed Cookie，一支影片是數百個 segment。當初防盜連的保護（見 `docs/s3-access-control-memo.md`）正是離線快取的天敵；要離線讀，得為「已下載內容」另闢不簽章或長簽章的路徑
2. **頁面全是個人化 SSR**。Dashboard 與閱讀頁依身份組資料，不能無腦快取；App Router 的 RSC payload 快取繁瑣，結論通常是動態頁全部 network-first——**離線時幾乎什麼都打不開**，除非刻意設計一套「離線書架」專屬儲存
3. **權限失效無法協調**。撤銷讀者、連結到期，離線快取不會知道。「被移除後立即失去存取」的產品承諾與「離線可讀」直接衝突，做之前要先決定哪個讓步

### 第三層：離線寫入＋上傳佇列（數週，動到資料流架構）

潛水場景真正要的是這層。

- 離線記錄本身可行：文字、語音 blob 存 IndexedDB
- **難點在寫入路徑改造**：現有流程假設「馬上有 server 回應」（新增頁面由 server 發 `_id`，後續操作都依賴它）。離線模式要改成 client 端臨時 ID＋佇列化 mutation＋回線 replay——是資料流架構的改造，不是加功能
- **補傳的平台差異**：Android 有 Background Sync（關掉頁面也補傳）；**iOS 完全沒有**，補傳只在 PWA 開在前景時發生。實務體驗是「回線後打開 app，看到『N 筆待同步』手動按同步」——及格，但不隱形

### 第四層：大檔背景上傳（PWA 天花板，做不到）

一小時課程影片數 GB，iOS 上 PWA 上傳中被切到背景就凍結——PWA 沒有背景傳輸，這是原生 `URLSession` 的專利。**這條線就是「該不該上 Capacitor」的分界**：

- 整堂課錄影（鋼琴家教情境）是真需求 → PWA 給不了合格體驗，評估 Capacitor（約 1–2 個月＋雙平台上架與長期維護兩個 build）
- 「文字＋語音＋幾張照片上岸補傳」（潛水情境） → PWA 第三層就夠

## 橫向的 iOS 特有風險

- **Cookie 隔離**：iOS 已安裝 PWA 與 Safari 是分開的儲存空間。使用者在 Safari 登入過，PWA 內仍是未登入，需在 PWA 內重新登入
- **OAuth 回跳**：登入流程若中途跳出到外部瀏覽器或 LINE app，callback 可能落回 Safari 而非 PWA，session 種錯地方。Google OAuth 在 standalone PWA 內一般可行；**LINE 登入（會彈去 LINE app）行為未知，是上線前最大的未驗證項**
- **儲存回收**：即使 `navigator.storage.persist()`，iOS 在空間壓力下仍可能清資料。「船上錄了三天還沒同步」有真實遺失風險——佇列 UI 必須醒目顯示未同步數量

## 建議路徑

| 層 | 工程量 | 建議 |
|---|---|---|
| 1 可安裝 | 1–2 天 | 隨時可做，做的時候順手實測 LINE OAuth（見下） |
| 2 離線快取讀 | 數週＋架構衝突 | **跳過**——這些場景痛的是離線「記」，不是離線「讀」 |
| 3 離線寫＋佇列 | 數週（動資料流） | 等潛水教練需求被驗證再做 |
| 4 大檔背景上傳 | PWA 做不到 | 真需求出現那天直接評估 Capacitor，不硬撐 PWA |
