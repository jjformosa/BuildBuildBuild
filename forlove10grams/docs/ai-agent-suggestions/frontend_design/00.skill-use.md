# Frontend Design Skills 使用指南

> 本文件整理已安裝的 frontend design skills 定位、差異與實際使用流程。

---

## Skill 全覽

### 通用型

| Skill | 類型 | 核心價值 |
|-------|------|----------|
| `frontend-design` (anthropics) | **行為指令型** | 強制 Claude 在寫 code 前先 commit 到一個大膽的美學方向，避免產出通用「AI slop」介面 |
| `ui-ux-pro-max` (nextlevelbuilder) | **工具型** | 透過 Python 腳本查詢 161 種產品類型、色盤、字型資料庫，產出精準的 design system |
| `redesign-existing-projects` (leonxlnx) | **流程型** | 專為既有專案設計：先審核 UI，再修 layout/spacing/hierarchy/styling |

### 風格特化型（確定走某風格才裝）

| Skill | 風格 | 對標產品 |
|-------|------|---------|
| `design-taste-frontend` (leonxlnx) | 可調三旋鈕（變異度/動態/密度） | 通用新專案 |
| `minimalist-ui` (leonxlnx) | 極簡 editorial | Notion、Linear |
| `high-end-visual-design` (leonxlnx) | 高端奢華，禁用 Inter/Roboto | Apple、Linear premium |
| `industrial-brutalist-ui` (leonxlnx) | 粗野工業，Swiss typography | 實驗性排版 |

**為什麼單獨使用效果有限：**
- `frontend-design`：prompt 沒給「目的/語氣/記憶點」三個錨點，Claude 會收斂到安全的中庸解
- `ui-ux-pro-max`：核心價值在 Python 腳本；若腳本沒執行，只用到文字說明，效果等同原生
- `redesign-existing-projects`：需搭配頁面截圖或程式碼，才能給出有針對性的審核

## 新 Next.js 專案流程

### Step 1：跑設計系統腳本，產出 MASTER.md

```bash
python3 .claude/skills/ui-ux-pro-max/scripts/search.py \
  "next.js saas [產品描述，例如: task management productivity minimal]" \
  --design-system --persist -p "專案名稱"
```

產出：
- `design-system/MASTER.md` — 風格、色盤、字型、anti-pattern 的設計聖經
- 後續所有頁面 prompt 都要 reference 此檔案

### Step 2：用 frontend-design 鎖定美學方向

給 Claude 這三件事，缺一不可：

| 問題 | 範例 |
|------|------|
| 這介面解決什麼問題？給誰用？ | 「給設計師用的 SaaS，需要感覺精緻但不沉重」 |
| 選一個極端的語氣（只選一個） | `editorial`、`brutalist`、`luxury`、`playful`... |
| 這頁的唯一記憶點是什麼？ | 「進入時有 hero 字型 reveal 讓人停住」 |

### Step 3：每次建頁面的標準 prompt 結構

```
請讀 design-system/MASTER.md，並確認是否有 design-system/pages/[page-name].md。

目標頁面：[Landing Page]
受眾：[...]
語氣：[editorial, 偏暗色調, 大字排版]
記憶點：[捲動時 hero 標題有視差拆字動畫]
技術棧：Next.js 15, Tailwind, Framer Motion

請先描述你選擇的美學方向，確認後再寫 code。
```

> **關鍵**：「先描述美學方向，確認後再寫 code」強迫 Claude 在產出前 commit 到明確方向。

## 既有 Next.js 專案改善流程

### Step 1：生成目標設計系統

```bash
python3 .claude/skills/ui-ux-pro-max/scripts/search.py \
  "你的產品描述" --design-system --persist -p "專案名稱"
```

### Step 2：審核現有頁面

**選項 A — 用 `redesign-existing-projects` skill（推薦）**：直接貼頁面截圖或程式碼給 Claude，skill 會自動引導審核→修復流程。

**選項 B — 手動用 ui-ux-pro-max**：

```
請參考 design-system/MASTER.md，
審核這個頁面並對照 ui-ux-pro-max Quick Reference §1–§6。

找出：
1. CRITICAL 等級問題（無障礙、觸控目標）
2. 讓頁面「感覺不精緻」的前三個視覺問題
3. 不需要大改但立即有感的 quick win

不要重寫整頁，先列清單。
```

### Step 3：分層改善

| 類型 | 範例 | 用哪個 skill |
|------|------|-------------|
| **Quick win**（< 10 分鐘） | 字型替換、間距修正、色彩 token 化 | ui-ux-pro-max §6 |
| **元件級** | 重寫 Card / Form / Button | frontend-design（單一元件） |
| **頁面級** | Landing / Dashboard 整頁重設計 | redesign-existing-projects 或兩個 skill 合用 |

### Step 4：元件重寫 prompt 範例

```
請讀 design-system/MASTER.md。

我要改善這個已存在的 <PricingCard> 元件（程式碼如下）。
不要改變資料結構和 props 介面，只改視覺和互動。

目標：符合 MASTER.md 的設計語言，
同時解決：灰色文字對比不足、hover 沒有回饋、間距不一致。

請先說明你打算改什麼、不動什麼，確認後再輸出 code。
```

## 最高 ROI 的三個優先動作

不確定從哪裡開始時，這三件事視覺提升最明顯：

1. **色彩 token 化** — 把散落的 hardcode hex 統一成 CSS variables，一次改全站
2. **Typography** — 換掉 Inter/system font，選一個有個性的字型
3. **Spacing rhythm** — 改成嚴格的 4/8px 倍數系統，消除「感覺不整齊」

## 驗證腳本是否正常運作

```bash
python3 .claude/skills/ui-ux-pro-max/scripts/search.py "saas productivity" --design-system
```

輸出應包含具體的顏色名稱、字型名稱、風格建議。若輸出為空或報錯，先確認 Python 版本：

```bash
python3 --version
```
