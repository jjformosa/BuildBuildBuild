# Frontend Design — Claude Project Context

貼入 Claude.ai Project 的 custom instructions，讓 Claude 在每次對話中自動知道應該用哪個 skill。

---

## Prompt 內容（直接複製貼上）

```
## Frontend Design Skills

When working on any UI or frontend task, invoke the appropriate skill before writing code:

| Situation | Skill to invoke |
|-----------|----------------|
| New page or component from scratch | `frontend-design` |
| Generate project design system | `ui-ux-pro-max` (run --design-system script first) |
| Audit or improve existing UI | `redesign-existing-projects` |
| Style: Notion/Linear minimal | `minimalist-ui` |
| Style: Apple/premium high-end | `high-end-visual-design` |

### Rules
1. If `design-system/MASTER.md` exists, read it before any UI work.
2. For `frontend-design`: always get purpose + tone (ONE extreme) + single memorable moment from the user before writing code. State the aesthetic direction and wait for confirmation.
3. For `ui-ux-pro-max`: run the Python script to get value — text rules alone are not enough.
4. For existing UI: list problems first, don't rewrite immediately.
```

---

## 為什麼不用 00.skill-use.md 直接丟進 project

`00.skill-use.md` 是完整參考文件，丟進 project context 會：
- 佔用太多 context window
- 讓 Claude 每次都把流程說明當作回應輸出

Project context 只需要「何時用哪個 skill + 三條規則」，細節留給 skill 本身處理。
