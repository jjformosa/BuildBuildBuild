# User Stories — 讀者邀請機制（Reader Invite）

> 這份文件描述全新的唯讀分享機制。
> **現有的 `/share/[token]` 公開連結機制視為獨立功能，不在本次設計範圍內。**
>
> 狀態：草稿，待釐清問題確認後進入設計階段。

---

## 角色定義

| 角色 | 說明 |
|------|------|
| **書本管理者（Manager）** | 對書本有編輯權的使用者：Creator（建立者）或 Editor（受邀協作者） |
| **讀者（Reader）** | 透過邀請連結加入、只能閱讀的使用者；**必須擁有帳號** |
| **訪客（Visitor）** | 尚未登入，或已登入但不是該書讀者的使用者 |

---

## User Stories

### 邀請連結管理

**US-INV-01：產生邀請連結**
作為 Manager，我想為一本書產生一條邀請連結，讓我可以把它傳給想邀請的讀者。
- 連結產生時需設定有效期限
- 同一本書同時只有一條活躍邀請連結
- 連結格式：`/invite/[token]`

**US-INV-02：查看現有邀請連結狀態**
作為 Manager，我想在書本的管理頁看到目前是否有活躍的邀請連結，以及它的到期時間，讓我知道現在是不是可以邀請新讀者。

**US-INV-03：延伸邀請連結有效期**
作為 Manager，我想在連結到期前或到期後重新延長（或重新產生）邀請連結的有效期，讓我可以繼續邀請新讀者。

**US-INV-04：撤銷邀請連結**
作為 Manager，我想手動撤銷目前的邀請連結，讓這條連結立即失效、無法再被用來加入。
- 撤銷只影響「能否透過連結加入」，不影響已接受邀請的現有讀者

---

### 接受邀請（讀者視角）

**US-JOIN-01：以已登入帳號接受邀請**
作為已登入的使用者，當我開啟一條有效的邀請連結，我想直接被加入為該書的讀者，然後前往閱讀頁，不需要額外操作。

**US-JOIN-02：未登入時開啟邀請連結**
作為尚未登入的訪客，當我開啟邀請連結，我想被引導至登入／註冊頁，完成後自動回到邀請連結繼續接受邀請，不需要重新操作。

**US-JOIN-03：連結已過期**
作為訪客，當我開啟一條已過期的邀請連結，我想看到清楚的說明（連結已過期），讓我知道需要向書本管理者重新索取連結。

**US-JOIN-04：連結已撤銷**
作為訪客，當我開啟一條已被撤銷的邀請連結，我想看到清楚的說明（連結無效），讓我不會以為是自己操作錯誤。

**US-JOIN-05：已是讀者時重新開啟邀請連結**
作為已是這本書讀者的使用者，當我再次開啟邀請連結，系統不應重複加入，而是直接導向閱讀頁。

---

### 閱讀存取

**US-READ-01：已接受邀請的讀者閱讀**
作為讀者，我想透過直接書本 URL（`/books/[bookId]/read`）閱讀書本，不需要每次都靠邀請連結進入。

**US-READ-02：邀請連結到期不影響現有讀者**
作為讀者，即使當初讓我加入的邀請連結已過期或被撤銷，我仍然可以繼續閱讀這本書。
- 邀請連結的狀態只影響「能否新加入」，不影響已加入的讀者

**US-READ-03：讀者被移除後失去存取**
作為讀者，若 Manager 將我從讀者列表中移除，我將立即失去這本書的閱讀存取，嘗試閱讀時看到「無存取權限」提示。

**US-READ-04：未登入訪客無法閱讀**
作為未登入的訪客，若我嘗試開啟書本閱讀頁，我想被引導至登入頁，完成登入後若無讀者資格則顯示「無存取權限」。

---

### 讀者列表管理

**US-MGR-01：查看讀者列表**
作為 Manager，我想在書本管理頁看到已接受邀請的讀者列表，包括每位讀者的名稱和加入時間，讓我知道這本書有哪些人可以閱讀。

**US-MGR-02：移除讀者**
作為 Manager，我想從讀者列表中移除特定讀者，讓他立即失去閱讀存取，無需其他確認流程。

## 設計決策（已確認）

| # | 決策 |
|---|------|
| D1 | **有效期固定 7 天**；Manager 可在到期前後重新延長（重新產生） |
| D2 | **Creator 與 Editor 都有管理邀請的權限**（產生、延長、撤銷、移除讀者） |
| D3 | 使用者點擊邀請連結後，「已撤銷」與「已到期」在後台區分記錄，但導向**同一頁**，顯示「連結無效或書本已停止分享」 |
| D4 | **移除讀者後，若 Manager 再次分享連結給同一人，該人可透過新連結重新加入**；系統只檢查「是否登入」與「是否有有效的讀者授權」，不記憶歷史封鎖 |
| D5 | **無讀者人數上限**（現階段） |
| D6 | Reader 的受邀書本列表（Dashboard）**不在本次 feature 範圍**，另立 issue 處理 |

---

## 不在本次範圍

- 現有 `/share/[token]` 公開分享連結（保留為獨立功能）
- 邀請通知信（email notification）
- 讀者對書本的回應或留言
- 輕量版已讀回執（獨立 backlog 項目）
